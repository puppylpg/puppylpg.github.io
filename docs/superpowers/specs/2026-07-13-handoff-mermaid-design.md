# 交接文档 Mermaid 配图设计

## 目标

在不改变交接文档原有信息与链接的前提下，补充少量能解释结构、流程和依赖关系的 Mermaid 图，让接手人先建立全局认知，再按链接下钻。

## 配图范围

共增加 5 张图：

1. **交接内容全景图**：放在正文开头，展示“自建独立、数据基建、性能、效果、稳定性、已交接”六个主题之间的关系。
2. **自建独立决策链**：放在“自建独立 / 架构 / 现状”，展示自建结果满足时直接返回、不满足时进入三方补充与融合的分支。
3. **数据基建流转图**：放在“数据基建”，将现有的文字箭头改为 AREC → AntLogs → Dataphin → 数仓处理 → DeepInsight 的流程图。
4. **性能观测闭环图**：放在“性能”，区分 AntMonitor 实时监控与日志数仓播报两条观测路径，并汇合到性能诊断。
5. **在线链路模块图**：放在“已交接 / 在线链路”，按模块职责展示 antragflow、base、自建 AREC、三方 AREC、结果合并与响应返回。

## 图形约束

- 全部使用兼容性较好的 `flowchart`，不依赖主题样式或外部资源。
- 每张图只解释一个视角，避免与相邻文字或其他图重复。
- 节点名称沿用文档中的业务术语；不新增未经文档支持的系统结论。
- 保留原有图片、链接、介绍和 Skill 清单，只新增图或将纯文本箭头替换为等价流程图。

## 验收标准

- 语雀正文中存在且能识别 5 个 `mermaid` 代码块。
- 原有语雀、AntMonitor、Darwin、DeepInsight 等链接全部保留。
- 5 张图分别落在预定章节，图形语义与相邻正文一致。
- 不修改文档标题、原有说明文字及链接目标。
