# 对话总结文章模板

## Frontmatter

```yaml
---
title: "核心结论或主题"
date: YYYY-MM-DD HH:MM:SS +0800
categories: [ai]
tags: [tag1, tag2, tag3]
description: "一句话摘要"
---
```

- `categories` 固定 `[ai]`，有更细分领域可追加如 `[ai, tools]`
- `tags` 从关键词中提取 3-8 个
- frontmatter 其他规则遵循仓库 CLAUDE.md「内容规则」

## 正文骨架

TOC 必须严格是下面这两行：

```markdown
1. Table of Contents, ordered
{:toc}

# 背景与目标

{用户最初的需求/问题是什么}

# 主要步骤

## 步骤一：{阶段标题}

{做了什么，用户提出了什么，AI 如何响应}

## 步骤二：{阶段标题}

{...}

# 核心结论

{学到了什么、解决了什么、最终成果}

# 参考

{如果有引用的文档、链接、命令等}
```

## 正文要求

- 不要大段粘贴原始对话，用第三人称叙述体重新组织
- 代码块、命令行等关键输出保留核心片段
- 突出"为什么这样做"和"结果是什么"
- 保留清晰原理阐述的结构化形式：层次化原理解释（执行链条、调用栈、架构层次对比等）保留层级列表、流程图或对比表格，不要为压缩字数改扁平
- 总长度控制在 500-2000 字，除非对话本身很长
